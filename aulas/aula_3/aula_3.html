Salvando sua vida seu cdigo

Se voc fechar o R, vai perder tudo que voc fez

Soluo: salve seu cdigo em um script!

Script um arquivo de texto sem formatao, com a extenso .R ou .r, ao invs de .txt. Pense no script como a receita, o R como a cozinha, e a anlise o jantar.

Voc pode editar no bloco de notas, Notepad++, Gedit,Vim, EMACS, etc, mas evite usar Word, ele insere formataes que atrapalham tudo

Use o RStudio!

Mas salvar onde?

O R usa uma coisa chamada diretrio de trabalho.

Ele encontra automaticamente tudo que esteja nesse diretrio e salva as coisas nesse diretrio. recomendado que cada projeto seu no R viva no seu prprio diretrio, e os scripts mudem o diretrio de trabalho para o diretrio da anlise logo no comeo

getwd() ##exibe o diretrio de trabalhoda sesso atual
## [1] "C:/Users/Vitor/Dropbox/disciplinas/Introducao ao uso da linguagem R/aulas/2017.2/Intro-Linguagem-R/aulas/aula_3"

Para mudar o diretrio de trabalho, use a funo setwd()

setwd("C:/Curso_R/aula1") ##muda o diretrio de trabalho da sesso atual

ATENO
O endereo deve vir sempre em aspas, e use / ao invs de \ no Windows

Cada coisa, um script!

Separe suas funes e anlises em scripts diferentes, vai facilitar muito sua vida.

Use nomes curtos e descritivos. Funcao1.R, funcao2.R, regressao.R, regressaoLogisticaMultivaridaAmostra1BTSverso1.r pedir pra sofrer.

Uma pasta para cada coisa:
~/doutorado/

- /manuscrito

Salvando tudo junto

Se voc quiser salvar tudo que est na memria do R, inclusive objetos use a funo save.image() com a extenso .RData

 save.image(file="tudoJunto.RData")

Isto til quando temos uma analise demorada, que gera objetos grandes, e queremos acessar o resultado dela rapidamente no futuro. Podemos tambm salvar objetos especficos em um .RData com a funo save():

save(file="resultadosSalvos.RData",list = list(resultado1, resultado2, resultado3))

Em geral, melhor salvar os scripts do que o workspace ou os objetos

Lendo arquivos do R

Abrir um arquivo, carregar um arquivo e executar um arquivo so coisas diferentes.

Abrir -> ver os contedos
- pode ser no editor do R, RStudio, ou externo
- no executa nem carrega nada

Executar -> rodar os comandos dentro do arquivo
- funo source()

Carregar -> colocar o contedo do arquivo para dentro de um objeto
- funes load(), read.table(), read.csv() e etc

Funo list.files()

Lista todos os arquivos no diretrio de trabalho. Pode ser usada para selecionar um grupo de arquivos que sero lidos sequencialmente, por exemplo, com o argumento pattern para selecionar arquivos que contenham uma determinada sequencia no nome

Ateno: apenas lista, no carrega nem abre os arquivos

list.files()
## [1] "aula_3.html" "aula_3.pdf"  "aula_3.R"    "aula_3.Rmd"
aulas = list.files(pattern = "aula") # arquivos que possuam "aula"" em qualquer lugar do nome
aulas
## [1] "aula_3.html" "aula_3.pdf"  "aula_3.R"    "aula_3.Rmd"
scripts = list.files(pattern = "\\.R$") # "\\.R$" significa arquivos terminados em .R
scripts # objeto contendo os nomes dos arquivos, no os contedos
## [1] "aula_3.R"

Lendo arquivos quaisquer

O R consegue ler arquivos externos, se voc disser a ele o que esperar

help("read.table")

read.table() especializada em ler tabelas em formato texto

Salvar os dados em formato texto facilita leitura, backup e compatibilidade com verses anteriores e posteriores de programas (.xls / .xlsx)
- arquivos de texto so entendidos automaticamente por software de versionamento (aula 9)

Existem algumas funes com opes padro que facilitam o uso da read.table()

read.csv() l arquivos usando como padro o ponto (.) como sepador de decimais e a vrgula (,) como separador de colunas.

read.csv2() usa a vrgula como separador de decimais e o ponto-e-vrgula (;) como separador de colunas

Existem pacotes especficos para ler e escrever .docx, .xlsx, .pptx, SPSS, SAS, shapefiles, e praticamente qualquer tipo de arquivo

Uma das principais habilidades para se usar o R encontrar o pacote certo pra fazer o que voc quer

Argumentos importantes

#argumentos so sempre separados por vrgulas na chamada da funo
#no comeo,  bom colocar cada argumento em uma linha para facilitar o entendimento
#TRUE habilita a opo, FALSE desabilita
arquivoLido =read.csv(file = "/arquivos/caixeta.csv", 
                  header = TRUE, #interpreta a primeira linha como sendo o nome das colunas
                  as.is = TRUE , # no altera interpreta o tipo de dado em cada coluna, no altera seus dados. Se for FALSE, texto  interpretado como factor
                  sep = ",", # define o caractere que deve ser o separador de colunas
                  dec = ".", #define o caractere que deve ser o spearador de decimal
)

O comando acima cria um objeto chamado arquivoLido, com o contedo de "arquivo.csv"

O arquivo original no alterado, no importa o que voc fizer dentro do R

Se voc quiser salvar suas alteraes ou o seus resultados em um arquivo de texto, use write.table(), write.csv() ou write.csv2()

write.csv(x= arquivoLido, #objeto a ser gravado
         file = "arquivoNovo.csv", #nome do arquivo que vai ser gerado
         sep = ",", # define o caractere que deve ser o separador de colunas
         dec = ".", #define o caractere que deve ser o spearador de decimal
         row.names = F #ignora nomes/numeros das linhas
)
